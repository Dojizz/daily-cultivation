# 日课回顾日历设计

## 目标

让用户在月度尺度上回答两类问题：

- 最近哪些天持续行动了？
- 某一项习惯每天投入了多长时间？

首版聚焦已经保存的日课记录，不改变数据库 schema。

## 页面结构

新增独立的“回顾”页面，包含月份切换和两个视图。

### 总览月历

- 每个日期格显示习惯完成数（例如 `2/3`）。
- 当日特质完成时显示独立印记。
- 后续可显示当天完成任务数，但不将任务作为首版的主视觉指标。
- 点击日期打开日期详情。

### 习惯单项月历

- 在顶部选择一个习惯，例如“吉他”。
- 有时长的日期显示分钟数，并以由浅到深的颜色表示投入量。
- 已打卡但时长为 0 的日期显示完成标记，和无记录区分。
- 无记录的日期不着色。
- 显示当月总时长、练习天数、最长连续天数和日均时长。

### 日期详情

- 展示当天全部习惯及其时长。
- 展示当天已完成的特质。
- 为后续接入任务完成情况和日记入口预留位置。

## 数据与边界

- `practice_records` 已存储 `practiceId`、当天零点 `date` 和 `durationMinutes`，足以支撑首版。
- 数据层需要增加按日期范围查询记录的接口；这不涉及 migration。
- 特质在首版只有“完成 / 未完成”两种状态，不引入评分。
- 历史上“当天应轮换到哪个特质”不作为首版展示依据；增删或重排特质会改变实时推导的历史结果。首版仅展示实际已保存的特质记录。

## 后续扩展

- 为特质记录加入每日 1–5 分自评，并在月历中以评分强度可视化。该功能需要新增持久化字段和 Room migration，实施前单独设计。
